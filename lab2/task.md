### Лабораторная работа 2
#### 1 Разработать два консольных приложения:

- **TCP-сервер** — принимает подключение одного клиента, получает и обрабатывает сообщения, отправляет ответы.

- **TCP-клиент** — подключается к серверу, отправляет сообщения пользователя и принимает ответы.

Передача данных должна выполняться **по TCP** с использованием **структуры сообщения фиксированного формата** (заголовок + данные).
#### 2 Формат сообщений
```C
#include <stdint.h>  
  
#define MAX_PAYLOAD 1024  
  
typedef struct  
{  
uint32_t length;  // длина поля type + payload  
uint8_t  type;  // тип сообщения  
char  payload[MAX_PAYLOAD]; // данные  
} Message;  
```
#### 3 Типы сообщений
```C
enum  
{  
MSG_HELLO  = 1, // клиент -> сервер (ник)  
MSG_WELCOME = 2, // сервер -> клиент  
MSG_TEXT  = 3, // текст  
MSG_PING  = 4,  
MSG_PONG  = 5,  
MSG_BYE  = 6  
};
```
#### 4 Требования к TCP-серверу
1. Создать TCP-сокет (`socket`).
2. Выполнить `bind` на порт.
3. Выполнить `listen(1)`.
4. Принять клиента (`accept`).
5. Получить сообщение `MSG_HELLO`.
6. Отправить `MSG_WELCOME`.
7. В цикле:
    - читать сообщение по структуре;
    - при `MSG_TEXT` выводить в консоль;
    - при `MSG_PING` отправлять `MSG_PONG`;
    - при `MSG_BYE` завершать соединение.
8. Обрабатывать разрыв (`recv = 0`).
9. Закрывать сокет.
#### 5 Требования к TCP-клиенту
1. Создать TCP-сокет.
2. Подключиться (`connect`).
3. Отправить `MSG_HELLO` (ник).
4. Получить `MSG_WELCOME`.
5. Обеспечить двусторонний обмен:
    - ввод пользователя → `MSG_TEXT`
    - `/ping` → `MSG_PING`
    - `/quit` → `MSG_BYE`
6. При получении:
    - `MSG_TEXT` — вывести;
    - `MSG_PONG` — вывести «PONG»;
    - `MSG_BYE` — завершить.
7. Завершаться при закрытии сокета.
