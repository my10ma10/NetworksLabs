# Лабораторная работа №3

## Многопоточный TCP-сервер с постоянными соединениями и широковещательной рассылкой

---

## 1. Цель работы

Расширить функциональность Лабораторной работы №2:

* Реализовать **многопоточный сервер** с использованием `pthreads`.
* Организовать **пул рабочих потоков**.
* Обеспечить **постоянные соединения** клиентов.
* Реализовать **широковещательную рассылку (broadcast)** сообщений.
* Добавить механизм корректного **отключения и переподключения клиентов**.

---

## 2. Теоретические сведения

### 2.1. Многопоточность

Для создания потоков используется библиотека `pthread`.

Основные функции:

* `pthread_create()` — создание потока.
* `pthread_mutex_lock()` / `pthread_mutex_unlock()` — защита общих данных.
* `pthread_cond_wait()` / `pthread_cond_signal()` — ожидание событий.
* `pthread_detach()` — отсоединение потока.

---

### 2.2. Пул потоков (Thread Pool)

Вместо создания нового потока для каждого клиента:

1. При старте создаётся фиксированное количество потоков (например, 10).
2. Основной поток выполняет `accept()`.
3. Новые соединения помещаются в **очередь заданий**.
4. Рабочие потоки берут соединения из очереди и обслуживают клиентов.

---

### 2.3. Широковещательная рассылка (Broadcast)

Сервер хранит список подключённых клиентов.
При получении `MSG_TEXT`:
* сообщение рассылается всем клиентам из списка.

---

## 3. Постановка задачи

На основе кода ЛР2 необходимо:

### 3.1. Модифицировать сервер

1. Создать пул из 10 потоков при запуске.
2. Организовать потокобезопасную очередь подключений.
3. Основной поток:

   * принимает подключения через `accept()`
   * помещает дескриптор клиента в очередь
4. Рабочий поток:

   * принимает `MSG_HELLO`
   * отправляет `MSG_WELCOME`
   * добавляет клиента в глобальный список
   * входит в цикл обработки сообщений:

     * `MSG_TEXT` → рассылка всем клиентам
     * `MSG_PING` → ответ `MSG_PONG`
     * `MSG_BYE` → удалить клиента и закрыть соединение
   * при разрыве соединения удаляет клиента из списка

---

### 3.2. Модифицировать клиент

1. После подключения отправлять `MSG_HELLO`.
2. Создать отдельный поток для приёма сообщений.
3. Главный поток:

   * читает ввод пользователя
   * отправляет `MSG_TEXT`
   * `/ping` → `MSG_PING`
   * `/quit` → `MSG_BYE`
4. При разрыве соединения:

   * автоматически выполнять переподключение с задержкой (например, 2 секунды)
   * повторно выполнять `HELLO/WELCOME`

---

## 4. Формат сообщения

Используется структура из ЛР2: 

```c
#define MAX_PAYLOAD 1024

typedef struct {
    uint32_t length; // длина поля type + payload  
    uint8_t  type; // тип сообщения  
    char     payload[MAX_PAYLOAD]; // данные  
} Message;
```

Типы сообщений:

```c
enum {
    MSG_HELLO = 1, // клиент -> сервер (ник)  
    MSG_WELCOME = 2, // сервер -> клиент  
    MSG_TEXT = 3, // текст  
    MSG_PING = 4,
    MSG_PONG = 5,
    MSG_BYE = 6
};
```

---

## 5. Алгоритм работы сервера

1. `socket()`
2. `bind()`
3. `listen()`
4. Создание пула потоков
5. Бесконечный цикл:

   * `accept()`
   * добавление клиента в очередь

Рабочий поток:

1. Получение сокета из очереди
2. Обработка `HELLO`
3. Отправка `WELCOME`
4. Добавление в список клиентов
5. Цикл приёма сообщений
6. Удаление клиента при отключении

---

## 6. Алгоритм работы клиента

1. `socket()`
2. `connect()`
3. Отправка `HELLO`
4. Получение `WELCOME`
5. Создание потока приёма
6. Основной цикл ввода
7. При разрыве — попытка переподключения

---

## 7. Ожидаемый результат

При запуске нескольких клиентов:

```
Client1 [ip:port]: Hello!
Client2 [ip:port]: Hi!
Client3 [ip:port]: Good evening!
```

Все сообщения отображаются у всех участников.

При отключении клиента:

```
Client disconnected: ClientX [ip:port]
```

Остальные клиенты продолжают работу.
